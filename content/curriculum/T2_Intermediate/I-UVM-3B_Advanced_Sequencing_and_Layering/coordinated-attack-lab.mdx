  ## Introduction

  In this lab, you will write a virtual sequence that coordinates two separate agents: a \`cfg_agent\` and a \`data_agent\`. You must ensure that the configuration is written via the \`cfg_agent\` *before* the data packets are sent via the \`data_agent\`.

  ## Objectives

  1. Create a virtual sequence class.
  2. Obtain handles to the target sequencers.
  3. Link the virtual sequencer with the individual agent sequencers in the environment's connect phase.
  4. Use a \`fork...join\` or sequential steps to correctly order the configuration and data sequences.

  ## Lab Task: Write the Virtual Sequence Body

  Examine the solution below to see how a \`coordinated_vseq\` manages its subordinate sequences.
