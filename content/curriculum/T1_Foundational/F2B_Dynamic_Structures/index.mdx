## Quick Take
- **What it is:** SystemVerilog collections that resize (`dynamic array`), preserve order (`queue`), or map sparse keys (`associative array`).
- **Why it matters:** Choosing the wrong structure wastes sim time, hides ordering bugs, or burns memory. The right choice keeps scoreboards and stimulus responsive when traffic spikes.
- **The Analogy:** Dynamic arrays are expandable trays, queues are conveyor belts, associative arrays are labeled drawers. Packed vs. unpacked tells you how the bits stack inside each drawer.
- **The Standard:** IEEE 1800-2023 §7.4–§7.10 (Arrays) and §7.12 (Array manipulation methods).

### Decision Matrix: Which Structure For What?

| Scenario | Best Choice | Why | Avoid |
|----------|-------------|-----|-------|
| Scoreboard tracking outstanding TXNs by ID | **Associative array** (keyed by ID) | Sparse, O(1) lookup by ID, auto-cleanup with `delete()` | Dynamic array (wastes memory for sparse IDs) |
| Driver stimulus sequence of packets | **Queue** | Preserves order, push/pop at ends, bounded for backpressure | Associative array (no ordering guarantee) |
| Coverage bin collection (known size) | **Dynamic array** | Fixed-size after init, O(1) access | Queue (unnecessary overhead) |
| Monitor collecting variable-length bursts | **Queue** | Unknown length, grows naturally | Dynamic array (requires explicit resize) |
| Protocol field lookup table | **Associative array** (keyed by enum/string) | Self-documenting, sparse | Queue (wrong abstraction) |

## Code Patterns You'll Use Daily

### Dynamic Array — Resize and Preserve

```systemverilog
// IEEE 1800-2023 §7.5.1 — dynamic array new[]
logic [7:0] buffer [];

initial begin
  buffer = new[4];           // Allocate 4 elements
  foreach (buffer[i]) buffer[i] = i * 10;

  buffer = new[8](buffer);   // Grow to 8, PRESERVE first 4 elements
  // buffer[0..3] = {0, 10, 20, 30}, buffer[4..7] = {0, 0, 0, 0}

  buffer = new[2](buffer);   // Shrink to 2, TRUNCATES
  // buffer = {0, 10} — elements [2..7] are LOST

  buffer.delete();            // Free all memory; buffer.size() == 0
end
```

> [!WARNING]
> **Common bug:** `buffer = new[8]` without the copy argument `(buffer)` allocates a fresh array and **discards all existing data**. Always pass the old array when growing.

### Queue — Bounded Backpressure

```systemverilog
// IEEE 1800-2023 §7.10
typedef struct { int id; bit [31:0] data; } txn_t;

txn_t pending_q [$:64];  // Bounded queue, max 64 entries

task enqueue(txn_t t);
  if (pending_q.size() >= 64) begin
    `uvm_warning("BACKPRESSURE", $sformatf("Queue full, dropping TXN id=%0d", t.id))
    return;
  end
  pending_q.push_back(t);
endtask

task dequeue(output txn_t t);
  wait (pending_q.size() > 0);  // Block until something arrives
  t = pending_q.pop_front();
endtask
```

> [!TIP]
> **Performance insight:** `push_front` and `push_back` are O(1). `insert(index, item)` is O(N) because it shifts all subsequent elements. In hot-path code (e.g., a scoreboard processing millions of TXNs), avoid mid-queue inserts.

### Associative Array — Transaction Tracking

```systemverilog
// IEEE 1800-2023 §7.8 — associative array with int key
class scoreboard extends uvm_component;
  txn_t expected_txns [int];  // Keyed by transaction ID

  function void add_expected(txn_t t);
    if (expected_txns.exists(t.id))
      `uvm_error("DUPE", $sformatf("TXN id=%0d already pending", t.id))
    expected_txns[t.id] = t;
  endfunction

  function void check_actual(txn_t actual);
    if (!expected_txns.exists(actual.id)) begin
      `uvm_error("UNEXPECTED", $sformatf("TXN id=%0d not in expected map", actual.id))
      return;
    end
    // Compare fields...
    expected_txns.delete(actual.id);  // Clean up after match
  endfunction

  function void report_phase(uvm_phase phase);
    if (expected_txns.num() > 0)
      `uvm_error("LEAK", $sformatf("%0d unmatched TXNs remain", expected_txns.num()))
  endfunction
endclass
```

> [!IMPORTANT]
> **Memory leak pattern:** If you never `delete()` matched entries, the associative array grows unbounded during long regressions, consuming gigabytes. Always clean up completed transactions.

### Array Manipulation Methods

```systemverilog
// IEEE 1800-2023 §7.12 — built-in array methods
int data_q [$] = '{3, 1, 4, 1, 5, 9, 2, 6};

// Reduction
int total = data_q.sum();        // 31
int product = data_q.product();  // 6480

// Searching
int indices [$] = data_q.find_index with (item > 4);  // '{4, 5, 7}
int first_big = data_q.find_first with (item > 4);     // 5

// Ordering
data_q.sort();                   // '{1, 1, 2, 3, 4, 5, 6, 9}
data_q.rsort();                  // '{9, 6, 5, 4, 3, 2, 1, 1}
data_q.shuffle();                // Random permutation
data_q.reverse();                // Reverse in place

// Unique
int uniq [$] = data_q.unique();  // Remove duplicates
```

## Deeper Understanding

### Performance Characteristics by Structure

| Operation | Dynamic Array | Queue | Associative Array |
|-----------|:---:|:---:|:---:|
| Random access `[i]` | O(1) | O(1) | O(log N) or O(1) amortized |
| `push_back` / `push_front` | N/A (must resize) | O(1) | N/A |
| `insert(mid)` | N/A | O(N) | N/A |
| `delete(key)` | N/A | O(N) | O(1) amortized |
| Resize / `new[N]` | O(N) copy | N/A | N/A |
| Memory overhead | Minimal | Moderate (linked list) | High (hash table) |
| `foreach` iteration | O(N) | O(N) | O(N) sorted order |

### Corner Cases Interviewers Love

1. **What happens if you pop from an empty queue?**
   The simulator returns the default value for the element type and may issue a warning (simulator-dependent behavior, not defined in the LRM). Always check `size() > 0` before popping.

2. **Can a dynamic array hold classes?**
   Yes. `my_class handles [];` is valid. But be careful — `new[N]` creates N null handles, not N objects. You must `new()` each element individually.

3. **What is the iteration order of an associative array?**
   When using `foreach` or `first()`/`next()`, iteration follows the key's sort order (ascending for int/string keys). This is guaranteed by IEEE 1800-2023 §7.8.

4. **Can you assign a queue to a dynamic array?**
   Yes — SystemVerilog allows assignment between compatible array types (IEEE 1800-2023 §7.4). `int da []; int q [$] = '{1,2,3}; da = q;` works and creates a dynamic array with 3 elements.

5. **What is a bounded queue and when should you use one?**
   Declared as `type q [$:N]`. It limits maximum size. Use it to model hardware FIFOs with fixed depth, or to add backpressure to testbench components. Pushing beyond the bound is implementation-defined — always check size first.

## Make It Work
- Pre-size dynamic arrays when you know the target (`new[size]`) to avoid silent copies during hot paths.
- Guard queues with bounds and logs so overflow becomes an intentional backpressure event, not a mystery performance cliff.
- Use associative arrays for sparse spaces (transaction IDs, addresses) and delete keys when work completes.
- Double-check packed vs. unpacked ordering before slicing signals for serializers or coverage bins.
