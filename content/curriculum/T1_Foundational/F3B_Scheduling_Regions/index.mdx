## Quick Take
- **What it is:** The IEEE standard divides each simulation time step into multiple scheduling regions (Active, Inactive, NBA, Observed, Reactive, etc.).
- **Why it matters:** It prevents race conditions between testbench code and RTL code, and between blocking and non-blocking assignments.

## The 4 Standard SV Scheduling Regions

SystemVerilog defines a strict order of execution within a single time step:

| Region | Primary Purpose | Examples |
|---|---|---|
| **Active** | Evaluate and update variables using blocking assignments, evaluate RHS of non-blocking assignments. | `a = b;`, `$display();` |
| **Inactive** | Process `#0` delaying statements (legacy, avoid using). | `#0 a = b;` |
| **NBA (Non-Blocking Assignment)** | Update LHS of non-blocking assignments evaluated in the Active region. | `a <= b;` |
| **Observed** | Evaluate concurrent assertions (SVA) using values sampled in the Pre-poned region. | `assert property (...)` |
| **Reactive** | Execute testbench program blocks, testbench checks. | Program block execution |
