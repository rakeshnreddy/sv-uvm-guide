## Quick Take
- **What it is:** The IEEE standard divides each simulation time step into multiple scheduling regions (Preponed, Active, Inactive, NBA, Observed, Reactive, Postponed).
- **Why it matters:** Without these regions, RTL and Testbench code would step on each other's toes, creating non-deterministic race conditions.
- **The Analogy:** Think of the time step as a factory assembly line. Parts are inspected (Preponed), assembled (Active), shifted down the line (NBA), QA tested (Observed), logged (Reactive), and shipped (Postponed).

## Build Your Mental Model

SystemVerilog defines a strict order of execution within a single time step to prevent race conditions (IEEE 1800-2023 §4). Here are the most critical regions you must understand for interviews:

| Region | Primary Purpose | Examples |
|---|---|---|
| **Preponed** | Sample stable values from the previous time step before any new changes occur. | Used by clocking blocks (input sampling) and concurrent assertions. |
| **Active** | Evaluate and update variables using blocking assignments, evaluate RHS of non-blocking assignments. | `a = b;`, `$display();` |
| **Inactive** | Process `#0` delaying statements (legacy, avoid using). | `#0 a = b;` |
| **NBA (Non-Blocking Assignment)** | Update LHS of non-blocking assignments evaluated in the Active region. | `a <= b;` |
| **Observed** | Evaluate concurrent assertions (SVA) using values sampled in the Preponed region. | `assert property (...)` |
| **Reactive** | Execute testbench program blocks and testbench checks. | Program block execution, action blocks of assertions |
| **Postponed** | Execute read-only tasks after all other regions have settled for the current time step. | `$monitor()`, `$strobe()` |

> [!IMPORTANT]
> **The Active vs. Reactive Split:** The original intent of SV scheduling was to isolate RTL (Active/NBA) from Testbench (Reactive). Testbenches driven from `program` blocks execute in the Reactive region, interacting with RTL via clocking blocks to guarantee zero race conditions. In modern UVM (which uses `module` and `class` constructs instead of `program`), this isolation is achieved purely via clocking block semantics and phasing.

## Make It Work
- **Avoid `#0` delays:** They were used in Verilog-1995 to fix races by pushing assignments to the Inactive region. In SystemVerilog, use non-blocking assignments (`<=`) instead.
- **Use `$strobe` for debug:** If you need to print a variable's final value at the end of a time step after all NBA updates, use `$strobe` (which executes in the Postponed region) instead of `$display` (which executes in the Active region).

## Push Further
- Investigate how clocking blocks span multiple regions: inputs are sampled in the Preponed region, and outputs are driven in the Re-NBA region (a subset of the Reactive region).
- Read the IEEE 1800-2023 LRM Section 4.4, which details the stratified event scheduler loop. Notice that the scheduler can loop back from NBA to Active if new events are triggered.
