## Quick Take
- **Functions:** Execute in zero simulation time. They cannot contain delays (`#`), event controls (`@`), or `wait` statements. Best for combinational logic and quick calculations.
- **Tasks:** Can consume simulation time. They can have delays and synchronization. Best for driving stimulus and complex verification sequences.
- **Lifetime (`automatic` vs `static`):** By default, tasks and functions in modules have `static` lifetimes, meaning their variables are shared across all calls. Use `automatic` to ensure each call gets its own memory space (re-entrant).

## Build Your Mental Model

### Functions (Zero Time)
Functions must execute immediately and return a result without blocking.

```systemverilog
// Function with a return value
function int add_nums(int a, int b);
  return a + b;
endfunction

// Void function (returns nothing)
function void print_status(string msg);
  $display("[%0t] Status: %s", $time, msg);
endfunction
```

### Tasks (Time Consuming)
Tasks can take their time. They are the backbone of testbench drivers and monitors.

```systemverilog
task drive_packet(input logic [31:0] data);
  @(posedge clk);
  enable <= 1'b1;
  bus_data <= data;
  @(posedge clk);
  enable <= 1'b0;
endtask
```

### The `automatic` Keyword: A Verification Lifesaver
In SystemVerilog modules, tasks and functions are `static` by default. This means local variables are allocated once and shared across all invocations. If two threads call the same task simultaneously, they will overwrite each other's data!

**Always use `automatic` in verification code to make tasks thread-safe (re-entrant).**

### Argument Passing
SystemVerilog supports multiple ways to pass arguments.

1. **Pass by Value (Default for inputs):** The task gets a copy of the variable.
2. **Pass by Reference (`ref`):** The task gets a pointer to the original variable. Changes are visible immediately. Highly efficient for large arrays. You must use `automatic` tasks/functions to use `ref`.
3. **Constant Reference (`const ref`):** Passes a pointer for efficiency, but makes it read-only.

```systemverilog
task automatic compute_crc(const ref byte payload[], output int crc);
  // payload is passed by reference (no copy overhead) but is read-only.
  // crc is an output.
endtask
```
