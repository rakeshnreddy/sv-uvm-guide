# Finding: canvas health check referenced a retired curriculum route

- Date: 2026-03-04
- Feature ID: FEAT-WEBGL-CANVAS-STABILITY-2026-03-04
- Severity: P2
- Status: closed

## Summary
The newly added Playwright canvas stability suite targeted a retired curriculum route (`/curriculum/T3_Advanced/A-SV-1_Advanced_Constraints/index`) for the `Constraint3D` visualizer. This made the shared Vitest curriculum route audit fail, blocking the QA acceptance gate even though the visualization itself had moved to `I-SV-2A`.

## Affected Area
- Files:
  - `tests/e2e/canvas-health-check.spec.ts`
  - `tests/qa/curriculumCoverageAudit.spec.ts`
- Modules/Lessons:
  - I-SV-2A constrained randomization fundamentals
  - Deprecated A-SV-1 advanced constraints route surface
- Test(s):
  - `npx vitest --run` (route-audit assertion path)

## Reproduction
1. Run `npx vitest --run`.
2. Observe `tests/qa/curriculumCoverageAudit.spec.ts` fail in `keeps hard-coded curriculum routes in Playwright specs aligned with current lesson paths`.
3. Inspect `tests/e2e/canvas-health-check.spec.ts` and find `/curriculum/T3_Advanced/A-SV-1_Advanced_Constraints/index` in `canvas3DPages`.

## Expected
Playwright curriculum route constants should only reference active curriculum lesson paths generated by the current curriculum map.

## Actual
The canvas-health route list referenced a removed route, causing the route-integrity QA audit to fail.

## Root Cause Hypothesis
The canvas-health suite was authored against an older curriculum structure and did not update the `Constraint3D` page path after module reorganization.

## Suggested Fix Direction
- Replace the retired advanced route with `/curriculum/T2_Intermediate/I-SV-2A_Constrained_Randomization_Fundamentals/index`.
- Keep route constants synchronized with `tests/qa/curriculumCoverageAudit.spec.ts` expectations whenever curriculum structures change.
