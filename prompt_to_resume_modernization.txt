**Established Patterns (follow exactly)**
Each new lesson follows this file structure:
1. **Lesson MDX**: `content/curriculum/T2_Intermediate/I-UVM-{N}_{Name}/index.mdx`
   - Structure: frontmatter (title, description, flashcards) → Quick Take → Build Your Mental Model → Interactive → Make It Work → Push Further → Practice & Reinforce (Quiz) → References & Next Topics
   - IEEE clause grounding is required
2. **Visualizer**: `src/components/visuals/{Name}Visualizer.tsx`
   - `"use client"` directive, dark theme (slate-900/800), lucide-react icons
   - Tabbed or interactive with state management
3. **Component Registration**: `src/app/curriculum/[...slug]/page.tsx`
   - Add `dynamic(() => import(...), { ssr: false, loading: () => <VisualizationFallback /> })` entry in the `components` map
4. **Flashcards**: `content/flashcards/I-UVM-{N}_{Name}.json`
   - Array of `{ id, question, answer }` objects
5. **Unit Tests**: `tests/components/visuals/{Name}Visualizer.test.tsx`
   - Vitest + React Testing Library; use `getAllByText`/`getAllByRole` for text that may match multiple elements
6. **Validation (mandatory before marking complete)**:
   - `npm run generate:curriculum`
   - `npm run lint`
   - `npx vitest --run` (expect snapshot updates for new curriculum entries — use `--update`)
7. **TASKS.md**: Update status in both the Active Backlog Summary table AND the Detailed Task Brief section, check all deliverable boxes.
